## Features

1. **Detail-view "Similar clips" sidekick** — `d76041a`
   - New `findSimilarClips(pivotId, { limit })` in lib/db. Scores
     clips by shared host (+4) and shared tags (+3 per, capped at
     9). Noise tags (image / link / text / url / long / redacted /
     scrubbed / quick-capture) filtered out so a "kind:image" pivot
     doesn't pull every other image.
   - Detail meta gets a new "Similar" row; each entry is a button
     with kind glyph + preview + reason pill ("@host" or "#N
     shared").
   - Race guard: if openDetail() steps to a different clip mid-
     flight, the older render drops itself.
   - Click any similar row → jumps to that clip's detail view.
   - CSS uses existing --surface/--accent vars with fallback rgba
     so it themes cleanly.

2. **"Purge >24h" trash quick action** — `9f80c02`
   - New button between Empty + the 7-day retention foot. Uses the
     existing `purgeOldTrash` helper exposed via a new
     `purgeTrashOlderThan` RPC.
   - Label live-counts qualifying rows ("Purge >24h (12)") and
     disables when nothing's old enough.
   - Cmd+K palette entry "Purge trash older than 24h" under Bulk.
   - New neutral `button.small` CSS so the pair visually balances
     with the danger-toned Empty without inheriting its red tone.

3. **Lang-aware code fence on copy-as-Markdown + export** —
   `e549438`
   - New `detectCodeLang(content)` in lib/util. Branch-by-branch
     heuristics covering json / yaml / sql / diff / markdown /
     python / go / rust / bash / css / html / jsx / typescript /
     javascript. Ordered so Go's `import "fmt"` doesn't read as
     Python and Rust's `let mut x: i32 = …` doesn't read as TS.
     Returns undefined on prose so the caller can fall back to a
     plain fence.
   - Bounded — only the first 4KB of the snippet is inspected.
   - Wired into popup.ts copyAsMarkdown + lib/export.ts toMarkdown
     + content.ts in-page palette Shift+Enter/Click path (which
     uses a small mirror `detectLangLite` to keep the content
     bundle from importing lib code).
   - 22/22 sanity tests: positives across 14 languages + 3
     negative cases (prose / empty / too-short).

4. **In-page palette per-host ranking** — `dc6c5b4`
   - Background attaches the active tab's host as `tabHost` on
     every `cc-open-palette` message (both context-menu and
     keyboard-shortcut paths).
   - openPalette sorts: pinned-first ALWAYS wins, then within
     each tier host-match floats up, then stable recency.
   - Matching rows carry a tiny "this site" badge (blue tint) and
     a faint blue divider separates the host-boost cluster from
     the rest inside the unpinned tier when both groups exist.
   - 8/8 sanity tests: empty + pinned-only + no-match passes
     through unchanged; github.com / example.com / docs.github.com
     / www.example.com all produce the exact expected ordering;
     boost works inside the pinned tier too.

5. **Per-host auto-scrub-origin site rule** — `f27c95b`
   - New `autoScrubOrigin?: boolean` on SiteRule (additive — no
     IDB version bump).
   - Background ingest: when matched rule sets the flag, wipes
     `item.source = {}` + pushes `scrubbed` tag BEFORE putClip.
     Tags + auto-redact + custom patterns still run BEFORE the
     scrub, so a (redact + scrub) rule produces a clip with the
     body masked AND the origin wiped.
   - Image previews that mentioned the dropped page get rewritten
     to a generic "Image · 800×600" so the title doesn't bleed
     through the preview label.
   - Popup settings: new "scrub origin" checkbox next to
     auto-pin / auto-redact / skip-capture. Form save / edit-
     load / reset all handle it. Rule list summary gets a "scrub"
     pill. Zero-effect guard updated.
   - 16/16 sanity tests: upsert round-trips the flag, list +
     findSiteRuleFor surface it, wildcard `*.partner.io` carries
     it, edit-mode preserves other fields, the pure scrub
     transform wipes source / keeps content / adds the tag
     idempotently, image preview rewrite fires when needed and
     stays out of the way when not.

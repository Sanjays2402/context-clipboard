## Features

1. **Quick-capture from system clipboard** — `61a9e4f`
   - New clipboard icon between save-search btn + note btn.
   - Tries navigator.clipboard.read first (rich, catches images);
     falls back to readText for older Firefox.
   - Tagged `quick-capture` via top-of-list re-read after addNote /
     addImageBlob (lastSeenAt-desc pins fresh row to idx 0).
   - Failure modes (empty / no permission / unsupported) surface as
     a one-line toast — no silent no-ops.
   - Surfaces in Cmd+K palette as "Capture from system clipboard"
     under Capture group.
   - Adds `clipboardRead` to both chrome + firefox manifests.
   - Three new icons: clipboard (with check), eraser, globe — used
     by this + the next two features.

2. **Per-clip "scrub origin"** — `3ce089c`
   - New eraser btn in detail header (between Redact + Pin) wipes
     source.url + source.title + source.nearbyText + source.favicon
     while keeping content + tags + pin + OCR + template flag.
   - Confirms with concrete loss before acting.
   - Tags `scrubbed`. Idempotent.
   - Re-opens detail after success so openDetail() stays the single
     source of truth for the panel layout.
   - Pure db helper `scrubClipOrigin(id)` for future bulk paths.
   - Surfaces in Cmd+K palette under Privacy (only available when
     a detail is open).

3. **Retroactive PII auto-redact** — `1e13e0b`
   - "Redact existing" btn in Settings (right below auto-redact
     toggle) + Cmd+K palette "Redact PII in every existing clip".
   - Sweeps every text clip, redacts ones whose body still contains
     PII via the same `redactPii` pipeline as on-capture.
   - REVERSIBLE — stashes `originalContent` so user can unmask any
     clip later from detail view. (Data was already on disk, no
     privacy cost to keeping a copy.)
   - Inline pre-count before confirm. "No PII found" toast when
     scan is clean.
   - New CSS row (.retro-redact-row + .retro-redact-label) keeps
     the btn inline with the toggle.
   - Sanity test: 9/9 retro-redact checks (candidate-selection
     predicate fires on email/phone/JWT/ghp_/sk-/Luhn-valid card,
     silent on plain text + non-Luhn digits, rewrite preserves
     surrounding chars + idempotent).

4. **Clip-row right-click menu** — `b762904`
   - Right-click any list row opens a compact action menu:
     Copy / Copy-as-MD / Open detail / Pin-toggle / Select-toggle /
     Tag / Filter-to-host / Forget-host / Trash.
   - State-aware labels: pin row flips Pin <-> Unpin, select row
     flips Add <-> Remove from selection, host rows hide entirely
     on scrubbed/origin-less clips.
   - Position-clamped to viewport (measure-then-nudge so menu
     doesn't spill off-screen).
   - Closes on outside mousedown / Esc / list scroll / window blur.
   - ~80 lines new CSS matches palette card visual (glass bg,
     danger-tint hover for trash/forget, monospace shortcut pills).
   - Cheatsheet picks up new "Right-click" row.
   - Forget-host integration pre-fills settings forget-host input
     and reuses runForgetHost() so the confirm flow stays identical
     to the settings path.

5. **In-page palette last-query memory** — `15f227c`
   - Cmd+Shift+V overlay pre-fills its search box with whatever the
     user last typed before closing.
   - lib/db gets get/setPaletteLastQuery (single meta row,
     trimmed + 200-char cap, empty/whitespace clears).
   - background reads it on every cc-open-palette dispatch (both
     context-menu Paste-from path + keyboard Cmd+Shift+V path),
     forwards as `lastQuery` in the message payload.
   - content's openPalette() pre-fills + select-alls so first
     keystroke replaces, Backspace / arrows resume editing.
   - content's closePalette() fires-and-forgets cc-rpc
     setPaletteQuery to persist (try/catch for runtime-torn-down
     edge case).
   - Works in side-panel mode too — single shared slot.
   - Sanity test: 9/9 palette last-query checks (in-process IDB
     shim implements just enough surface to round-trip — initial
     empty, round-trip, trim, 200-char cap, empty/whitespace
     clear, re-set after clear, undefined input).

   - 26/26 autoLock sanity + 78/78 existing site-rules-io still
     green

3. **Detail-view `lockedAt` breadcrumb** — `47d43bf`
   - new optional `lockedAt?: number` on ClipItem records the
     false→true lock transition; cleared on unlock so re-lock
     starts fresh — answers "when did I decide this is
     irreplaceable?" not "when have I EVER locked this"
   - new "Locked" meta row hidden unless `locked === true` AND
     `typeof lockedAt === "number"` (back-compat: pre-stamp
     clips simply hide row)
   - db.toggleLock + setLocked stamp on transition; setLocked
     no-op fast path skips BOTH write AND lockedAt mutation
   - background autoLock spreads `{ locked: true, lockedAt: now }`
     on fresh path; dedup-path stamps ONLY on transition
     (preserves original timestamp for repeat hits)
   - NEW pure `lib/locked-since.ts` with 6-tier formatLockedSince
     returning `{label, tooltip}`: <1m / <1h / <24h / yesterday /
     2-6 weekday / 7+ ISO. Tooltip ALWAYS the absolute "YYYY-MM-DD
     HH:MM" so hover reveals exact moment. Defensive: negative
     ages clamp, missing/NaN fall back to "Locked"
   - 30/30 sanity covers all 6 tiers + defensive shapes +
     boundary precision (59s vs 60s, 23h vs 24h, last-night 23:59
     in hour tier not yesterday, day-6 vs day-7 ISO crossover) +
     tooltip always-absolute contract + clock zero-padding

## Notes per slice

1. **Day-count badge** — new `lib/day-group.computeDayHeaderInfos` returns
   `{label, count}` per run-start (counts the contiguous run forward);
   `computeDayHeaders` is now a thin label-only projection of it so the
   two can't drift. Muted, lighter-weight count span after the label.
   Pinned + undated tiers get their run-count too.
2. **Lightbox** — new pure `lib/lightbox` (`canZoom` gate +
   `lightboxCaption` mirroring the detail image-info line to the digit).
   Overlay at z-index 50 (above detail/settings/cheatsheet). Esc /
   backdrop / × close; `closeDetail` tears it down so navigating away
   never strands it; close drops the (multi-MB) data URL from memory.
   Image never blurred in the lightbox even under anti-shoulder-surf.
   Enter/Space on the focused image opens it (role=button).
3. **Tag reorder** — new pure `lib/tag-chips.reorderTags` (move from->to
   with before/after edge, cleaning+dedupe, no-op guards return the
   cleaned list). HTML5 DnD, same drop-edge model + CSS classes
   (`dragging` / `drop-before` / `drop-after`) as the saved-search and
   recent-search strips. Commits through the same `updateTags` + render
   path the chip × and raw input use.
4. **is:langoverride** — new `lang-override.hasLangOverride` (known lang
   id OR the `OVERRIDE_NONE` sentinel — same values `selectValueFor`
   treats as non-Auto). Presence-only, direction-agnostic. Wired through
   parser / applyQuery / describeQuery / Cmd+K Filter (live count, greys
   at 0) / empty-state hint. `langOverrideCount` cache refreshed on the
   same `wide` pass as `wrapOverrideCount`.
5. **Export fence lang** — new `lang-override.exportFenceLang` (forced
   lang wins; `"none"` -> `""` bare fence; auto -> detected). New
   `markdownAsFence` in popup decides fence-vs-prose so a forced LANGUAGE
   forces a fence even when `looksLikeCode` wouldn't fire, and forced-OFF
   forces a prose blockquote. `send-to.fencedCodeForClip` honors it too
   via a new optional `SendableClip.langOverride`, threaded through both
   `buildSendActions` call sites (render-time + dispatch-time agree).
